# WUST Algo Frontend

WUST Algo Frontend 是 WUST ACM 算法训练数据平台的前端项目，基于 Vue 3、TypeScript 和 Vite 构建。项目用于展示 OJ 绑定、刷题动态、个人资料、团队、消息、数据统计与后台管理等功能。

后端仓库：`https://github.com/everlastingQAQ/WUST-Algo-tracker`

## 功能概览

- 首页统计：总 AC、今日/本周/本月/今年数据，以及 AtCoder、牛客、洛谷、CodeForces、QOJ 平台拆分统计。
- OJ 绑定：支持 AtCoder、NowCoder、LuoGu、CodeForces、QOJ。
- 个人资料：头像链接、邮箱通知、密码修改、团队信息、成员刷题数据、训练画像、成就徽章预览与详情抽屉、最近 30 天总结、各平台题数/提交数和 OJ 同步状态。
- 团队系统：创建团队、邀请成员、编辑团队名称和头像、队长管理成员、转移队长、队员退出团队、队长解散团队，并提供团队训练看板。
- 消息页：默认展示私信会话，支持发送私信、未读提醒、系统消息和团队邀请处理入口。
- 数据统计：公开统计页、用户排名、团队排名、提交趋势图。
- 训练画像：根据近期提交、AC 增长、AC 率、最后提交时间、深夜提交比例和平台覆盖自动生成训练标签、简短总结和建议。
- 抓取队列可视化：手动更新 OJ 数据后展示排队中、抓取中、完成和失败；后台可查看近期抓取任务。
- 后台管理：用户管理、角色/分组调整、密码重置、抓取任务查看、教练/管理员群发站内消息、系统邀请码配置。
- 主题与布局：支持多主题、响应式统计页、统一页脚。
- 访问控制：未登录用户不可查看动态、比赛和排名数据；统计概览仍可公开查看。

## v1.1.2 更新

v1.1.2 聚焦团队权限闭环和成就体验打磨：

- 团队权限：队长可转移队长、解散团队；普通队员可退出团队。
- 团队管理：个人资料团队面板支持“设为队长”“移除成员”等队长操作，普通成员只看到符合权限的按钮。
- 成就体验：成就预览、详情抽屉、筛选按钮和关闭按钮统一为网站标准按钮字体、圆角和尺寸。
- 版本展示：左下角状态栏版本号更新为 `v1.1.2`，便于区分本轮功能。

## v1.1 能力

v1.1 聚焦训练反馈和团队氛围，新增三个前端规则型模块，均基于真实提交数据和现有统计接口生成，不使用假数据：

- 成就徽章：根据总 AC、多平台覆盖、WA 后 AC、CE、深夜提交、团队贡献占比、团队 WA 占比、站内提交/AC 率分位和本机改密码次数等真实行为自动点亮徽章；个人页只展示精选预览，点击“查看全部”打开详情抽屉，未解锁隐藏成就不暴露条件，所有已触发成就都会在本机永久保留解锁状态。
- 团队训练看板：在个人资料页团队卡中展示团队本周 AC、本月 AC、活跃成员、本周贡献条和本周火力位。
- 训练周报：位于首页热力图下方，展示近 7 天 AC、提交、活跃天数、较上周变化、主要活跃平台和规则型训练建议。

当前徽章和周报使用 `src/utils/v11Features.ts` 生成，AC 口径为 `platform + problem` 去重；当题目名为空时回退到提交 ID，跨平台同名题不会合并。

## 当前功能边界

当前版本不包含“两人数据对比”页面，左侧栏没有数据对比入口，个人资料页也不提供“与 TA 对比”按钮。若访问 `/compare`，前端会进入通用 NotFound 页面。

```bash

```

```bash

```

仅预览构建产物：

```bash

```

## 部署

部署脚本默认配合后端仓库的 `/opt/wust-algo` 目录结构：

```text
/opt/wust-algo/
├── frontend/
├── tracker/
├── bin/
├── conf/
└── infra/
```

准备配置：

```bash
cd /opt/wust-algo/frontend
cp deploy/.env.example deploy/.env
nano deploy/.env
```

关键配置：

- `DOMAIN`：站点域名或 IP。
- `NGINX_PORT`：Nginx 监听端口。
- `USER_UPSTREAM`：用户服务地址，默认 `127.0.0.1:8000`。
- `CORE_UPSTREAM`：核心数据服务地址，默认 `127.0.0.1:8001`。
- `BACKEND_UPSTREAM`：后端 gateway 地址。

执行部署：

```bash
bash deploy/scripts/deploy-frontend.sh
```

脚本会执行依赖安装、生产构建、渲染 Nginx 配置并 reload Nginx。

## 后端接口约定

生产环境 Nginx 会把前端请求反代到后端：

- `/api/user/*` -> 用户服务 `/v1/user/*`
- `/api/core/*` -> 核心数据服务 `/v1/core/*`
- `/api/agent/*` -> AI 总结服务 `/v1/agent/*`

前端会对密码进行 SHA-256 后提交，后端存储和校验的也是哈希值。

## 私信与消息

消息入口位于左侧栏“消息”。默认进入“私信”页签，可搜索用户并发起会话；“系统消息”页签保留团队邀请和系统通知。

前端使用的私信接口：

- `GET /api/user/message/conversations`：会话列表。
- `GET /api/user/message/thread?userId=<id>`：与指定用户的聊天记录。
- `POST /api/user/message/send`：发送私信。
- `POST /api/user/message/read`：标记会话已读。
- `GET /api/user/message/unread-count`：未读数量。
- `POST /api/user/message/broadcast`：教练和管理员群发站内消息。

首页消息卡会显示最近私信提醒；个人资料页查看他人主页时提供“发消息”入口。

## 团队权限

团队入口位于个人资料页左侧团队卡片。

- 无团队用户可创建团队，创建者自动成为队长。
- 队长可编辑团队名称和头像、邀请无团队成员、移除非队长成员、将队长转移给其他成员。
- 队长可解散团队；解散后所有成员变为无团队，待处理团队邀请会失效。
- 普通队员可在自己的个人资料页点击“退出团队”；队长不能直接退出，需要先解散团队。
- 查看其他用户个人资料时，只展示团队信息和成员数据，不展示管理按钮。

## 页面访问规则

- 未登录访问 `/allActivities` 时显示“登录后可以查看动态”，不请求动态接口。
- 未登录访问 `/contest` 或 `/contest/:id` 时显示“登录后可以查看比赛”，不请求比赛接口。
- 未登录访问 `/statistics` 时保留基础统计和趋势图，但隐藏用户排名和团队排名。
- 登录后可查看动态、比赛、排名，并可使用私信功能。

## 训练状态标签

个人资料页会显示训练画像、最多 4 个训练状态标签、规则型建议和最近 30 天总结；数据统计页的用户排名会显示一个核心状态。标签由前端根据真实数据自动计算：

- `稳定训练中`：最近保持提交和 AC。
- `爆发上升中`：本周或本月 AC 相比上一周期增长明显。
- `疯狂补题`：近期提交密度较高。
- `WA 战神`：提交很多但 AC 比例偏低。
- `夜间训练型`：最近提交中深夜比例较高。
- `多平台开荒`：近期覆盖多个 OJ 平台。
- `疑似掉队`：多天无提交或暂无提交记录。
- `观察期`：数据不足以判断明显状态。

当前提交日志没有题目难度和题单完成字段，因此不会伪造“1700+ 高难突破”或“题单完成”状态；后续如果后端补充题目难度、标签、题单数据，可在 `src/utils/trainingStatus.ts` 中扩展规则。

最近 30 天总结示例：

```text
近 30 天 AC 45 题、提交 120 次、活跃 18 天，比上一个 30 天增加 12 题，主要活跃于 NowCoder 和 CodeForces。
```

## OJ 数据同步状态

个人资料页会展示已绑定 OJ 的平台统计和抓取状态。每个平台会显示：

- AC 题数。
- 提交数。
- 最近成功同步时间。
- 同步状态。

前端展示文案：

- `已同步`：最近一次成功抓取仍在可信窗口内。
- `未同步`：从未成功抓取、数据可能过期或最近一次抓取失败。
- `抓取中`：当前正在抓取该平台。

点击“更新 OJ 数据”后，页面会根据后端返回的 `jobId` 轮询任务状态，并展示当前平台和完成进度。后台用户管理页的“抓取任务”区块可查看最近任务，支持按全部、排队、抓取中、失败筛选。

## 常用维护命令

```bash
npm run build
sudo nginx -t
sudo systemctl reload nginx
```

部署后可验证：

```bash
curl -I http://127.0.0.1:8088/
curl -I http://127.0.0.1:8088/allActivities
curl -I http://127.0.0.1:8088/contest
curl -I http://127.0.0.1:8088/statistics
```

由于 Vue Router history fallback，`/compare` 可能仍返回静态站点 `200`，但页面内容应由前端路由渲染为 NotFound，且构建产物中不应包含数据对比入口。

如果更新后浏览器仍显示旧页面，请强制刷新或清理浏览器缓存。
